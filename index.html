Procedularly Generated Dungeon

One of two current works in progress, this program is destined to become a roguelike dungeon crawl set in a cyberpunk dystopian city. For now, its main accomplishment is randomly generated dungeons, complete with sprites. It creates rooms and halls via random generation and a simple maze-building algorithm I’m fond of, then connects them with doorways. After cleaning up, it displays the dungeon using Java’s built-in library.

My next focus is adding gameplay elements, currently you have basic control of a character, but it’s clunky. I’m hoping to smooth out the controls and add attacks and skills, and once that’s finished change the refresh rate to a standard one, instead of every time the program accepts input, this will allow me to animate sprites and such. I have plans for a final-fantasy-esc combat system as well, but that’s further in the future.

The most difficult challenge I face currently with this program is I chose to build it from scratch, rather than use a game engine. This allows me much more in-depth control, but also means I have to build almost every facet from scratch, which takes time. However, I’m enjoying the extra layer of challenge and the experience is teaching me a lot about how game engines and similar programs work behind the scenes.

D&D Assistant

Dungeons and Dragons is one of my more elaborate past-times. Unfortunately, one I rarely get to indulge in. It can be intimidating to newcomers; the rules are complex. I wanted to build a system that could guide someone without any prior knowledge through the basics of character creation and gameplay. The result was an intricate, text-based program that will eventually step-by-step you through character development, while also allowing for the customization that makes D&D so interesting to me.

The downside to my current design for this project is currently, the enormous amount of data that needs to be parsed through. Other programs like this are often built as websites, with an SQL database to back them up, but I wanted this program to be more personal. Currently I’m toying with the idea of hard data stored in files alongside the program, that could be read and edited by the program, but not seen anywhere else. This would allow for maximum customization, as well as the preservation of homebrew data without having to make an account. Perhaps in certain circumstances, these files could also be shareable, to allow Dungeon Masters to share homebrew with their players and vice versa.

.BMP to Black & White

A deceptively simple program, meant to accept a .bmp image and produce a copy of the image but in black and white. Not by copying and editing the file in any way, but by transcribing the bits that make up the original, hex group by hex group, into a newly created file. Editing the information as the program goes, so the resulting file is the same image, but in black and white.

An experiment in learning about how data is stored on computers, and how it can be accessed and changed in unusual ways when security protocols are lax. Other parts of this project included sifting through scrambled data files for useful information and manipulating files without the editing being shown in the metadata.

This project has one major flaw in that, somewhere, there is a significant memory leak. It functions fine on small images, but when testing on larger images, the process consumes so much memory that most reasonable operating systems abort it automatically, resulting in the copy being mostly empty. However, when allowed to finish, the program does do exactly what it promises. If I had the time to improve this project, it would primarily involve finding that memory leak and stemming it, as well as searching for ways to make the entire program more time and memory efficient.

Version Error

After four years studying computer science in the hopes of becoming a game designer, I still had yet to touch a real game engine. I decided to change that. My capstone project for my Bachelor’s in computer science, a fairly simple game in Unity. My goal was to learn as much as possible, without going too deep into any one area, as I had limited time. I made a list of gameplay mechanics I hoped to create and gave myself a week to implement each one to the best of my ability. While doing so I learned about Unity’s physics and lighting engines, taught myself C#, built an entire playground for my little capsule player to run around in, added mechanics such as deadly darkness, item collection, and implemented a clever narrator using code from Unity’s store. I even designed an AI for patrolling enemies to watch for you, chase you, and search the area they lost you in for a few moments, before returning to their patrol. I was particularly proud of that. It made use of Unity’s built-in pathfinding, as well as some basic ray-tracing to determine if you were in the enemies line of sight.

The plot of the game was as shallow as the rest of it. You were testing a game for your friend, explaining the lack-luster graphics and still-present bugs, but the AI designed to keep the horror game ‘scary’ was out of control and had captured your friend somewhere in the test level. It was more an excuse, if anything, but I liked it enough to name the game after it, v.ERROR.

This project taught me a lot about game design that I never would have expected. I was lucky to not only have my professor and the internet as references, but a friend who was experienced with Unity and also using it to make a game.

AI Maze Solver

In an attempt to learn more about agent-based artificial intelligence, I repurposed a maze-generating algorithm from another project and designed an agent to explore the maze, hunt down jewels, kill monsters and find an exit in as few moves as possible. It utilizes both breadth-first and depth-first search algorithms, and even has the potential to include a fog-of-war effect.

Learning about artificial intelligence was enlightening. I’ve always been of the philosophy that computers are only as intelligent as the people who instruct them. Many believe artificial intelligence to be evidence to the contrary, but I think it’s only more evidence in my favor. For example, while working on this project, I noticed an interesting tendency for my agent to prioritize moving northwest, even if it wasn’t necessarily the most efficient path. Eventually I discovered that was because the line of code where the agent examined its choices for movement, listed moving north and west as the first options, so, if it was possible, the agent would always move north and west. The program was no smarter than I could make it.

Meal Planning Website

It’s no secret I have ADHD, and that makes some things harder. Like eating regularly and planning meals. I wanted to build a website that would help me, and others with the same issue, by making it as easy as possible to plan meals and keep track of ingredients.

The site uses an SQL database to keep track of ingredients, recipes, and meal plans. It also ascribes them to private, password-protected accounts, unless the user specifies they’d like to make them public.

There are forms for adding ingredients, creating recipes out of them, and then randomly or manually adding those recipes to a meal plan. The database can store meal plans or rules for randomly generating meal plans (vegetarian options only at dinner, for example) and will automatically compile a list of all the ingredients you’ll need for the week. If you want, you can even include prices for your ingredients and set a budget. Meal plans and grocery lists are also easily transcribed to PDFs to make sticking to your plan even easier.

My favorite part of this project, though, was going way overboard with the CSS and JavaScript. Even though the focus of the project was the database, I always enjoy making my work look nice. I built custom radio and check boxes, designed SVG logos, and while I ran out of time to implement it, laid the groundwork for a dark mode. Aesthetics are hardly ever the most important part of software, but it doesn’t take much effort to take a simple project from decent to amazing, if you’re willing to put in the time.
